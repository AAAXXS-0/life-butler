# 07-主动服务

> 本文档汇总各 Agent 的主动服务触发场景
> 触发方式：Heartbeat（每整点 cron run）
> 更新时间：2026-06-06

---

## 1. Coordinator 主动服务

| 场景 | 动作 |
|------|------|
| 每整点（heartbeat） | 检查 emergency_events |
| 触发 2 个子 Agent 检查 | cron run Schedule/Account 主动服务（**Trip 不需要**） |

---

## 2. ~~Trip Agent 主动服务~~（不需要）

**Trip Agent 不需要主动服务**。唤醒路径只有两条：
1. **mockend 异常检测器**（anomaly_detector.js）—— 写 diff 到 inbox + cron run 唤醒
2. **coordinator 委托**—— 用户请求行程规划

详见 `ARCHITECTURE/05-Mock-Backend.md` §9.3。

## 3. Schedule Agent 主动服务

| 场景 | 动作 |
|------|------|
| alarm 触发 | 发送提醒消息 |
| habit 长期未执行 | 询问是否删除或继续 |
| 纪念日/生日 | 提前 3 天提醒 |
| 行程结束同步 | trips.json 同步为 schedule 碎片 |

---

## 4. Account Agent 主动服务

| 场景 | 动作 |
|------|------|
| 当月超预算 | 提醒用户 |
| Trip 行程结束 | 主动询问：旅途花费要不要补录 |
| 7天无记录 | 询问是否需要补录 |

---

## 5. 触发链

```
每整点（00, 01, 02, ... 23）
    ↓
Coordinator 巡检 cron job 触发
    ↓ cron run
Trip 检查 / Schedule 检查 / Account 检查
    ↓ 各 Agent 内部判断
    ↓ 异常 → 通过 butler-comm 通知 Coordinator
    ↓
Coordinator 决定是否告知用户
```

---

## 6. 特殊主动服务

### 6.1 Mock Backend 事件触发的 replan

```
Mock Backend cron run（每 30 分钟）
    ↓ 生成 event
    ↓ 通知 Coordinator
    ↓
Coordinator 判断 severity
    ↓
    ├─ low：记录，不通知用户
    ├─ medium：通知 Trip Agent 自动切备选，事后告知用户
    └─ high：通知用户确认后强制 replan
```

### 6.2 遗忘清单主动服务

```
Schedule Agent 偶尔查看 memory-seven-dim-skill 遗忘清单
    ↓
联合时间规律判断用户是否空闲
    ↓
    ├─ 空闲：生成提醒话术，通知 Coordinator
    └─ 不空闲：不打扰
```
